# Skills

Reusable skill documents providing structured workflows for common engineering tasks.

## Skills

| Skill | When to Use |
|---|---|
| `frontend.md` | Building/modifying UI components with shadcn/ui + Tailwind |
| `backend.md` | Building/modifying API routes with rate limiting |
| `ui-review.md` | Reviewing UI for responsiveness, a11y, theme consistency |
| `architecture.md` | Making architecture decisions, analyzing data flow |
| `playwright-testing.md` | Writing or running E2E tests |
| `debugging.md` | Systematically debugging issues across the stack |
| `git.md` | Git operations, commits, PRs |
| `deployment.md` | Building and deploying the application |
| `performance.md` | Profiling and optimizing performance |
| `security.md` | Security review and vulnerability assessment |

Each skill follows the same structure: Purpose → When to Use → Workflow → Checklist → Best Practices → Common Mistakes → Exit Criteria.
