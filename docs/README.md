# Documentation

Engineering and product documentation for Scout AI.

## Structure

```
docs/
├── api/           — API route specifications
├── architecture/  — System architecture, data flow diagrams
├── decisions/     — Architecture Decision Records (ADRs)
├── features/      — Feature specifications and behaviors
├── reports/       — Project reports and sample outputs
├── demos/         — Demo scripts and walkthroughs
├── testing/       — Testing strategy and plans
└── setup/         — Setup and configuration guides
```

## Conventions

- ADRs follow standard format: Title, Status, Context, Decision, Consequences, Alternatives
- Feature docs include: purpose, flow, edge cases, implementation files
- API docs include: endpoint, method, request/response shapes, error codes, rate limits
